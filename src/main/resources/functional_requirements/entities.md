# Entity Specifications

## Overview
This document defines the entities required for the book data analysis application that retrieves data from the Fake REST API, performs analytics, and generates weekly reports.

## Entities

### 1. Book Entity

**Purpose**: Represents book data retrieved from the Fake REST API (https://fakerestapi.azurewebsites.net/index.html)

**Attributes**:
- `bookId` (Long): Unique identifier from the API (maps to "id" field)
- `title` (String): Book title
- `description` (String): Book description
- `pageCount` (Integer): Number of pages in the book
- `excerpt` (String): Brief excerpt from the book
- `publishDate` (LocalDateTime): Publication date of the book
- `retrievedAt` (LocalDateTime): Timestamp when the data was retrieved from API
- `analysisScore` (Double): Calculated popularity/analysis score (nullable)
- `reportId` (UUID): Reference to the report this book is included in (nullable)

**Relationships**:
- Many-to-One with Report: Multiple books can be included in one report
- One-to-Many with AnalyticsJob: One analytics job can process multiple books

**Notes**:
- The entity state is managed by the workflow system and represents the processing state (e.g., "extracted", "analyzed", "completed")
- `bookId` is the business identifier from the external API
- `analysisScore` is calculated during the analysis phase
- `reportId` is set when the book is included in a report

### 2. Report Entity

**Purpose**: Represents generated analytics reports containing insights about books

**Attributes**:
- `reportId` (String): Business identifier for the report (e.g., "REPORT-2024-W36")
- `reportType` (String): Type of report (e.g., "WEEKLY_ANALYTICS")
- `generatedAt` (LocalDateTime): When the report was generated
- `reportPeriodStart` (LocalDateTime): Start of the reporting period
- `reportPeriodEnd` (LocalDateTime): End of the reporting period
- `totalBooksAnalyzed` (Integer): Total number of books included in analysis
- `totalPageCount` (Long): Sum of all page counts
- `averagePageCount` (Double): Average pages per book
- `popularTitles` (String): JSON array of popular book titles
- `publicationDateInsights` (String): JSON object with publication date analysis
- `reportSummary` (String): Generated summary text
- `emailRecipients` (String): Comma-separated list of email addresses
- `emailSentAt` (LocalDateTime): Timestamp when email was sent (nullable)
- `analyticsJobId` (UUID): Reference to the analytics job that generated this report

**Relationships**:
- One-to-Many with Book: One report can include multiple books
- Many-to-One with AnalyticsJob: Multiple reports can be generated by one analytics job

**Notes**:
- The entity state is managed by the workflow system (e.g., "generated", "email_sending", "delivered")
- `reportId` is a human-readable business identifier
- `popularTitles` and `publicationDateInsights` store JSON data for flexibility
- `emailSentAt` is null until the email is successfully sent

### 3. AnalyticsJob Entity

**Purpose**: Represents scheduled data extraction and analysis jobs

**Attributes**:
- `jobId` (String): Business identifier for the job (e.g., "JOB-2024-W36-WED")
- `jobType` (String): Type of job (e.g., "WEEKLY_DATA_EXTRACTION")
- `scheduledFor` (LocalDateTime): When the job is scheduled to run
- `startedAt` (LocalDateTime): When the job actually started (nullable)
- `completedAt` (LocalDateTime): When the job completed (nullable)
- `booksProcessed` (Integer): Number of books processed (default: 0)
- `reportsGenerated` (Integer): Number of reports generated (default: 0)
- `errorMessage` (String): Error message if job failed (nullable)
- `nextJobId` (String): Reference to the next scheduled job (nullable)
- `configurationData` (String): JSON configuration for the job

**Relationships**:
- One-to-Many with Book: One analytics job can process multiple books
- One-to-Many with Report: One analytics job can generate multiple reports

**Notes**:
- The entity state is managed by the workflow system (e.g., "scheduled", "running", "completed", "failed")
- `jobId` follows a pattern that includes week and day information
- Jobs are automatically scheduled for every Wednesday
- `configurationData` stores job-specific settings as JSON
- `nextJobId` creates a chain of scheduled jobs

## Entity Relationships Summary

```
AnalyticsJob (1) -----> (Many) Book
AnalyticsJob (1) -----> (Many) Report
Report (1) -----> (Many) Book
```

## State Management Notes

All entities have their state managed by the workflow system through `entity.meta.state`. The states represent the processing lifecycle:

- **Book states**: extracted → analyzed → completed
- **Report states**: generated → email_sending → delivered  
- **AnalyticsJob states**: scheduled → running → completed (or failed)

These states are not part of the entity schema but are accessible via `entity.meta.state` in processor code.
