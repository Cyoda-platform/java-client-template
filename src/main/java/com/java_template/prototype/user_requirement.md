Build a Java application to power the Lovable UI using only Cyoda’s standard REST APIs (/entity/<Entity> CRUD + optional SQL reads). No custom endpoints. Functionality: product list → cart → checkout (user profile with inline address) → dummy payment auto-approves after 3s → order created (auto-split shipments) → picking loop (audit) → waiting to send → sent → delivered. Entities: Product, Cart, User, Payment, Order, Shipment, Reservation, PickLedger. Workflows: CartFlow, IdentityFlow, PaymentFlow, OrderLifecycle. Orchestration: User browses Products, Adds items, Cart edits, Checkout, Payment, Order, Picking updates, Shipments marked SENT then DELIVERED. API Surface: Products, Cart, User, Payment, Order/Shipment/Picking. Processors: Cart, Identity, Payment, Order, Fulfillment, Background. Guards: Cart must not be empty at checkout. Reservations must be ACTIVE. Address is always taken from User and snapshotted into Order. ULID required for Order.orderNumber.
